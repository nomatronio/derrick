## About the Server Workflow

When you deploy your app with Derrick, your computer (the Derrick client) connects
to the Derrick server to initiate the Derrick workflow. The client then registers
itself as a runner, allowing Derrick to run builds on it.

When the build is completed, the Derrick server sends the build artifacts, such as a container image,
to the artifact registry of your choice. Multiple Derrick clients can concurrently
connect to a single server to deploy and release applications.
